Cómo hacer Ping

Diagnóstica problemas de conectividad y latencia utilizando el comando ping

En Windows:

  1. Presiona Win + R
  2. Escribe cmd y presiona Enter
  3. Ejecuta: ping google.com
  4. O prueba con tu gateway: ping 192.168.1.1

En Mac/Linux:

  1. Abre Terminal
  2. Ejecuta: ping google.com
  3. Para detener: presiona Ctrl + C

Interpretación de resultados:

Bueno: Tiempo < 50ms, 0% pérdida de paquetes
Aceptable: Tiempo 50-100ms, pérdida < 5%
Problemático: Tiempo > 100ms o pérdida > 5%

Comandos útiles:

ping -t google.com Windows: Ping continuo
ping -c 10 google.com Mac/Linux: 10 pings
ping 8.8.8.8 Ping a DNS de Google

Cómo hacer Traceroute

Rastrea la ruta que toman los paquetes hasta su destino para identificar problemas de routing

En Windows:

  1. Presiona Win + R
  2. Escribe cmd y presiona Enter
  3. Ejecuta: tracert google.com
  4. Espera a que termine (puede tardar 1-2 minutos)

En Mac/Linux:

  1. Abre Terminal
  2. Ejecuta: traceroute google.com
  3. En algunos Linux: tracepath google.com

Interpretación de resultados:

Cada línea representa un "salto" en la ruta:

  • Salto 1: Tu router/gateway (192.168.x.x)
  • Saltos 2-4: Equipos del ISP local
  • Saltos 5+: Internet backbone

Señales de problema:

* * * Request timed out

El salto no responde (puede ser normal en algunos routers)

Tiempos muy altos (>200ms)

Posible congestión en ese punto de la red

Se detiene en salto específico

Problema de routing o bloqueo en ese punto

Cuándo usar traceroute:

  • Sitios específicos no cargan pero otros sí
  • Conexión lenta a ciertos destinos
  • Para reportes de IP bloqueada
  • Diagnóstico de problemas de routing

Diagnóstico de Red Avanzado

Comandos adicionales para diagnóstico completo de problemas de red

Verificar configuración de red:

ipconfig /all Windows: Configuración completa
ifconfig Mac/Linux: Configuración de red

Limpiar caché DNS:

ipconfig /flushdns Windows: Limpiar DNS
sudo dscacheutil -flushcache Mac: Limpiar DNS

Probar resolución DNS:

nslookup google.com Verificar resolución de nombres
nslookup google.com 8.8.8.8 Probar con DNS específico

Test de Velocidad por Terminal

Realizar tests de velocidad desde línea de comandos para diagnósticos precisos

Instalación (una sola vez):

curl -s https://install.speedtest.net/app/cli/install.deb.sh | sudo bash Ubuntu/Debian
brew install speedtest-cli Mac con Homebrew

Uso básico:

speedtest Test completo automático
speedtest --accept-license --accept-gdpr Primera ejecución
